# BankApp (Vite + Vanilla JS)

Небольшое учебное банковское веб-приложение с авторизацией, историей операций и базовыми действиями по счёту.

## Что умеет приложение

- Вход по `username` + `PIN`
- Просмотр баланса
- Просмотр операций (депозиты/списания)
- Сортировка операций
- Перевод денег между аккаунтами
- Запрос кредита
- Закрытие аккаунта

## Технологии

- `Vite`
- `JavaScript (ES Modules)`
- `HTML/CSS`

## Требования

- `Node.js` 18+ (рекомендуется актуальная LTS)
- `npm`

Проверка версий:

- `node -v`
- `npm -v`

## Установка и запуск локально

1. Перейти в папку проекта:
   - `cd BankApp`

2. Установить зависимости:
   - `npm install`

3. Запустить dev-сервер:
   - `npm run dev`

4. Открыть адрес из терминала (обычно):
   - `http://localhost:5173/`

## Сборка и предпросмотр production

- Сборка:
  - `npm run build`
- Локальный предпросмотр сборки:
  - `npm run preview`

## Как тестировать (чек-лист)

### 1) Авторизация

1. Войти под `ja / 1111`
2. Ожидаемо:
   - Появляется приветствие `Welcome back, James`
   - Открывается основная панель

### 2) Перевод

1. Войти под `ja / 1111`
2. В блоке **Transfer money** указать:
   - `Transfer to`: `ec`
   - `Amount`: `100`
3. Ожидаемо:
   - В списке операций появляется списание `-$100.00`
   - Баланс уменьшается на `$100.00`

### 3) Кредит

1. Под тем же пользователем запросить кредит `1000`
2. Ожидаемо:
   - В операциях появляется депозит `$1,000.00`
   - Баланс увеличивается

### 4) Закрытие аккаунта

1. В блоке **Close account** ввести:
   - `Confirm user`: `ja`
   - `Confirm PIN`: `1111`
2. Ожидаемо:
   - Приложение скрывается (разлогин)
   - Повторный вход `ja / 1111` больше недоступен

### 5) Негативные сценарии

- Неверный логин/пин → сообщение `Wrong username or password!`
- Клики по перевод/кредит/закрытие **без логина** не должны ломать приложение
- Перевод самому себе не выполняется
- Перевод при недостаточном балансе не выполняется

## Полезные npm-скрипты
